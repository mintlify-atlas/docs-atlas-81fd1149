title: "Quickstart"
description: "Build and run your first Kissflow custom component in five steps."
---

<Note>
  The SDK can only be initialized when running inside the Kissflow platform as an embedded custom component (iframe context). Calling `KFSDK.initialize()` outside of Kissflow will return a rejected Promise.
</Note>

<Steps>
  <Step title="Scaffold a project">
    Use the `create-kf-component` CLI to generate a ready-to-use project skeleton. Run the following command and follow the prompts to choose a framework and project name:

    ```bash
    npm create kf-component
    ```

    The CLI requires Node.js `>=18.14.1`. It creates a minimal frontend project pre-wired for Kissflow with a dev server, build config, and a sample entry point.
  </Step>

  <Step title="Install the SDK">
    Inside your new project directory, install the SDK:

    ```bash
    npm install @kissflow/lowcode-client-sdk
    ```

    If you scaffolded with `create-kf-component`, the SDK is already listed as a dependency — just run `npm install`.
  </Step>

  <Step title="Initialize the SDK">
    Import the SDK and call `KFSDK.initialize()`. Initialization is asynchronous and returns a Promise that resolves to the `kf` object.

    ```javascript
    import KFSDK from '@kissflow/lowcode-client-sdk'

    let kf
    ;(async function () {
        kf = await KFSDK.initialize()
    })()
    ```

    After this call, `kf` is fully set up and every namespace (`kf.context`, `kf.app`, `kf.client`, etc.) is available.
  </Step>

  <Step title="Read user info and interact with form fields">
    Access the authenticated user's details directly from the `kf.user` object:

    ```javascript
    const { Name, Email, _id, AppRoles } = kf.user
    console.log(`Logged in as ${Name} (${Email})`)
    ```

    Read and update form fields using `kf.context`:

    ```javascript
    // Read the current value of a field
    const currentValue = await kf.context.getField('fieldId')
    console.log('Field value:', currentValue)

    // Update one or more fields
    await kf.context.updateField({
        fieldId: 'new value',
        anotherFieldId: 42,
    })
    ```
  </Step>

  <Step title="Show a UI toast">
    Use `kf.client.showInfo()` to display a toast notification inside the Kissflow UI:

    ```javascript
    kf.client.showInfo('Component loaded successfully!')
    ```

    You can also prompt the user with a confirmation dialog:

    ```javascript
    const action = await kf.client.showConfirm({
        title: 'Confirm action',
        content: 'Are you sure you want to proceed?',
    })

    if (action === 'OK') {
        // user confirmed
    } else {
        // user cancelled
    }
    ```
  </Step>
</Steps>

## What's next?

Now that your component is running, explore the full SDK surface:

- Use `kf.app.getVariable` / `kf.app.setVariable` to read and write app-level state
- Use `kf.app.openPage(pageId)` to navigate between pages
- Use `kf.context.getTable(tableId)` to work with form tables and their rows
- Use `kf.formatter` to convert raw strings to Kissflow-typed values
- Use `kf.api(url, config)` to make authenticated HTTP calls
